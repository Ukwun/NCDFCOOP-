# Coop Commerce - Screenshot Capture Guide for PlayStore

---

## OVERVIEW

This guide provides detailed specifications for capturing 8+ screenshots for Google Play Store submission. Screenshots are critical for app store optimization (ASO) - they influence 65-80% of download decisions.

**Document Date:** February 17, 2026  
**Target Screenshots:** 8-12 device types (phone + tablet)  
**Dimensions:** Optimized for Google Play Store requirements  
**Timeline:** Capture Days 3-4 of Phase 1, before submission

## 1. PLAYSTORE SCREENSHOT REQUIREMENTS

### Google Play Store Specifications
```
Minimum Screenshots Required:
- 2 screenshots mandatory (very basic minimum)
- 8-12 screenshots recommended (industry standard)
- Max file size per screenshot: 20 MB (usually 2-5 MB)
- Format: PNG or JPEG
- Safe area: Must avoid UI elements outside 2% margin

Device Types to Show:
Android phones (required):
- 4.7" phone (720 √ó 1280 px) - Standard dimension
- 5.5" phone (1080 √ó 1920 px) - Large phone (RECOMMENDED primary)

Android tablets (optional but recommended):
- 7" tablet (1280 √ó 1920 px) - Small tablet
- 10" tablet (1440 √ó 2560 px) - Large tablet

Platform Requirement:
- All screenshots must be from Android device/emulator
- iOS screenshots NOT accepted by Google Play (iOS only)
- Portrait orientation only (landscape not supported)
```

### PlayStore Asset Arrangement
```
Screenshot Ordering (Left-to-Right Swipe):
1. First Screenshot: HERO/MAIN VALUE PROPOSITION
   - What does this app do? (Most critical)
   - Shown as store listing preview
   - Highest conversion impact

2. Second Screenshot: Key Feature #1
3. Third Screenshot: Key Feature #2
4. Fourth Screenshot: Key Feature #3
5. Fifth Screenshot: Key Feature #4
6. Sixth Screenshot: Member/Loyalty Benefits
7. Seventh Screenshot: Purchase/Checkout Flow
8. Eighth Screenshot: Call-to-Action
   - Download, Rate, Join, etc.
   - Final impression before decision

Recommended Total: 8-10 screenshots (not more than 10)
```

## 12. SCREENSHOT CAPTURE WORKFLOW

### Pre-Capture Preparation (30 minutes)
```
1. Device Setup (Android Emulator or Real Device)
   - API Level: 28 or higher (Android 9+)
   - Screen Resolution: Match 1080 √ó 1920 px if emulator
   - System Language: English
   - Time Zone: Lagos/GMT+1
   - Time: Set to ~9:00-14:00 (reasonable time)
   - Battery: Full or near-full
   - WiFi: Connected
   - Screen Timeout: 5+ minutes
   - Developer Mode ON (if real device)
   - USB Debugging ON (if real device)

2. App Environment
   - Firebase/backend connection: Stable
   - Test data loaded: Full product catalog visible
   - Member account: Prepared with test data
     * Member tier: Gold or Platinum for best visuals
     * Points: 1,000+ for visible balance
   - Products: Variety of prices, ratings, images
   - Orders: Some completed orders for tracking screenshots
   - Network: Good connection (no loading delays)

3. Screenshot Tool
   - Device native: Volume Down + Power (Android)
   - OR: Android Studio built-in screenshot tool
   - OR: adb shell (command line)
     adb shell screencap -p /sdcard/ss.png && adb pull /sdcard/ss.png
   - Save location: Dedicated folder (screenshots_v1.0.0/)
   - File naming: 01_hero_home.png, 02_product_browsing.png, etc.

4. Testing Checklist Before Capture
   - [ ] App loads without crashing
   - [ ] All text is readable (no cutoff)
   - [ ] Images are visible (product images, icons)
   - [ ] Colors match brand guidelines (Brand Green visible)
   - [ ] No loading spinners or placeholders
   - [ ] All buttons are accessible and visible
   - [ ] Navigation is intuitive
   - [ ] Member benefits are evident
   - [ ] Pricing is clear and accurate
   - [ ] Time/date stamp is realistic
```

## 16. TROUBLESHOOTING COMMON SCREENSHOT ISSUES

```
Issue: Text appears blurry or pixelated
Solution:
- Ensure emulator resolution is exactly 1080 √ó 1920
- Use Pixel 4a or similar device (1080 √ó 2340 for newer phones)
- Real device: Screenshot is sharp by default
- Check "Scale display" setting (should be automatic)

Issue: ProductImages not loading or showing placeholders
Solution:
- Ensure good WiFi connection
- Wait 5 seconds after navigating to screen
- Force reload: Pull-to-refresh (if available)
- Restart emulator/app if persistent
- Check Firebase connectivity in app

Issue: Ratings or member data not showing
Solution:
- Use test account with complete profile
- Add products to cart to create transaction history
- Check that Firebase is initialized (check logs)
- Ensure mock fallback data is available

Issue: "Add to Cart" button not visible (cut off)
Solution:
- Make sure screen height is 1920px (full height)
- Scroll up/down slightly to center the button
- Try different emulator: Pixel 5 (1960px height) may help
- Crop screenshot to exact 1920px if needed

Issue: Member loyalty card doesn't show member tier
Solution:
- Log in with an account that's actually a member
- Generate test data via Firebase Console
- Ensure member tier is set: Bronze/Silver/Gold/Platinum
- Check that Firestore has member role in auth claims

Issue: PlayStore says image dimensions invalid
Solution:
- Check file metadata: identify filename.png (ImageMagick)
- Must be exactly 1080 √ó 1920 px
- Re-export from original source if needed
- Use ImageMagick to verify/fix:
  identify 01_hero_home.png
  convert input.png -resize 1080x1920 output.png

Issue: Screenshot is too large or uploads slowly
Solution:
- Compress PNG: pngquant --quality 70-85 input.png
- Or use Online tool: tinypng.com, compressor.io
- Target: Keep under 2 MB per image
- 8 images √ó 2 MB = 16 MB total (well under PlayStore limit of 160 MB total)
```
