PAYMENT FLOW ARCHITECTURE
═════════════════════════════════════════════════════════════════════════════

CARD PAYMENT FLOW:
═══════════════════

1. Customer selects "Card Payment"
   ↓
2. FlutterwavePaymentButton shows payment method selection
   ↓
3. Customer clicks "Pay ₦X.XX"
   ↓
4. FlutterwavePaymentService.initiatePayment() called:
   - Saves payment record to Firestore
   - Calls Flutterwave API: POST /v3/payments
   - Returns authorization URL
   ↓
5. Payment link opens in browser/WebView
   ↓
6. Customer enters card details in Flutterwave hosted page
   ↓
7. Payment processed by Flutterwave
   ↓
8. Flutterwave sends webhook: POST /onFlutterwaveWebhook (charge.completed)
   ↓
9. flutterwave-webhook.js handler:
   - Verifies signature
   - Updates payment status = 'completed'
   - Updates order status = 'confirmed' (IMMEDIATELY)
   - Deducts inventory atomically
   - Notifies customer + sellers
   ↓
10. Order is CONFIRMED immediately ✅

═══════════════════════════════════════════════════════════════════════════

BANK TRANSFER PAYMENT FLOW:
═══════════════════════════

1. Customer selects "Bank Transfer"
   ↓
2. FlutterwavePaymentButton shows bank transfer confirmation
   ↓
3. BankTransferDetailsDialog displays:
   - Bank name
   - Account name
   - Account number
   - Reference code (ORD-{orderId})
   - Transfer instructions
   ↓
4. Customer clicks "I've Transferred the Money"
   ↓
5. Payment status shows "Pending Verification"
   ↓
6. Customer completes bank transfer to specified account
   ↓
7. Bank confirms transfer
   ↓
8. Flutterwave receives transfer and sends webhook: POST /onFlutterwaveWebhook
   ↓
9. flutterwave-webhook.js handler:
   - Verifies signature
   - Updates payment status = 'completed'
   - Detects payment_type = 'bank_transfer'
   - Updates order status = 'payment_pending_verification' (NOT confirmed yet)
   - Deducts inventory atomically (RESERVED)
   - Notifies customer + sellers (even though pending)
   ↓
10. Admin/Seller reviews transfer in dashboard
    ↓
11. Admin clicks "Verify Bank Transfer"
    ↓
12. Cloud Function: verifyBankTransfer.js called
    ↓
13. Function updates:
    - payment.verificationStatus = 'verified'
    - order.status = 'confirmed'
    - Sends final confirmation to customer
    ↓
14. Order is CONFIRMED ✅

FIRESTORE DATABASE STRUCTURE
═════════════════════════════════════════════════════════════════════════════

Collection: payments
├── Document: {paymentId}
│   ├── orderId: string
│   ├── reference: string (Flutterwave reference)
│   ├── amount: number
│   ├── currency: "NGN"
│   ├── status: "completed|failed|pending"
│   ├── paymentType: "card|bank_transfer"
│   ├── verificationStatus: "unverified|verified" (for bank transfers)
│   ├── verifiedBy: string (admin user ID)
│   ├── verificationTimestamp: timestamp
│   ├── metadata: {...}
│   └── createdAt: timestamp

Collection: orders
├── Document: {orderId}
│   ├── status: "confirmed|payment_pending_verification|payment_failed"
│   ├── paymentType: "card|bank_transfer"
│   ├── paymentStatus: "completed|failed"
│   ├── paymentMethod: enum
│   ├── transactionId: string
│   ├── items: array<{...}>
│   ├── deliveryAddress: {...}
│   ├── total: number
│   └── ...other fields

Collection: transactions (Audit Trail)
├── Document: {transactionId}
│   ├── orderId: string
│   ├── type: "payment_created|payment_verified|payment_webhook"
│   ├── status: "success|failed"
│   ├── amount: number
│   ├── paymentMethod: string
│   ├── verifiedBy: string (if manual verification)
│   ├── notes: string
│   └── timestamp: timestamp

API ENDPOINTS
═════════════════════════════════════════════════════════════════════════════

Flutterwave API Base URL: https://api.flutterwave.com/v3

1. INITIATE PAYMENT:
   POST /payments
   Body: {
     tx_ref: "ORD-{orderId}",
     amount: 1000,
     currency: "NGN",
     redirect_url: "https://your-app.com/payment-callback",
     customer: {
       email: "customer@example.com",
       name: "John Doe",
       phonenumber: "+234..."
     },
     payment_options: [
       "card",   // Credit/Debit card
       "account" // Bank transfer
     ]
   }

2. VERIFY PAYMENT:
   GET /transactions/{id}/verify
   Returns payment status and details

3. PROCESS REFUND:
   POST /refunds
   Body: {
     transaction_id: 123456,
     amount: 1000
   }

Cloud Function Endpoints:

1. WEBHOOK (Automatic):
   POST /.../onFlutterwaveWebhook
   Triggered by: Flutterwave when payment completes

2. VERIFY BANK TRANSFER (Callable):
   Call from admin dashboard with:
   {
     orderId: string,
     paymentReference: string,
     verificationNotes: string (optional)
   }

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: Webhook not firing
Solution:
   1. Verify webhook URL is correct in Flutterwave dashboard
   2. Check Firebase functions logs: firebase functions:log
   3. Test webhook manually using Flutterwave test mode

Problem: Bank transfer not showing in Firestore
Solution:
   1. Check payment metadata in Flutterwave dashboard
   2. Verify payment_type is detected correctly in webhook
   3. Check Cloud Function logs for errors

Problem: Bank transfer details not showing to customer
Solution:
   1. Ensure getBankTransferDetails() is called in checkout service
   2. Update bank account details in BankTransferDetailsDialog
   3. Verify dialog opens after bank transfer selection
