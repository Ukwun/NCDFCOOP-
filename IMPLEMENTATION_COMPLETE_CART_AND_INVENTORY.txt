## 🔄 HOW IT WORKS

### User Journey 1: Shopping with Cart Persistence

```
Day 1: App Opens
  ↓ Cart initializer runs
  ↓ Firebase loads user session
  ↓ App calls initializeCart()
  ↓ Cart loads from Firestore
  ✅ Previous cart items appear

User browses Rice product
  ↓ Stock level = 25
  ↓ Green message: "✅ In Stock"
  ↓ Selects qty: 2
  ✅ Button enabled: "Add to Cart"

User adds to cart
  ↓ Item saved to Firestore
  ↓ Activity logged

User closes app

Day 2: App Opens Again
  ↓ Cart initializer runs
  ↓ initializeCart() loads from Firestore
  ✅ Rice (qty 2) still in cart!
```

### User Journey 2: Inventory Validation at Checkout

```
User has 3 items in cart:
  • Rice (qty 2) - 5 in stock ✅
  • Oil (qty 1) - 0 in stock ❌
  • Beans (qty 3) - 3 in stock ✅

Clicks "Complete Order"
  ↓ System validates inventory:
  ↓ validateCartInventory() runs
  ↓ Checks each item:
    - Rice: need 2, have 5 ✅
    - Oil: need 1, have 0 ❌ FAIL
    - Beans: need 3, have 3 ✅
  ↓ Validation fails

Error shown: "⚠️ Some items no longer available"
  ↓ User can't proceed
  ↓ User adjusts cart (removes Oil)
  ↓ Tries checkout again
  ↓ All items validate ✅
  ↓ Order created successfully
  ↓ Stock deducted:
    - Rice: 5 > 3
    - Beans: 3 > 0
```

## 💡 TECHNICAL DETAILS

### Inventory Status Colors
| Status | Stock | Color | Icon |
|--------|-------|-------|------|
| Available | 10+ | Green (#10B981) | ✅ |
| Low | 5-9 | Amber (#F59E0B) | ⚠️ |
| Critical | 1-4 | Red (#EF4444) | 🚨 |
| Out of Stock | 0 | Dark Red (#DC2626) | ❌ |

### Validation Flow at Checkout
```
User clicks "Complete Order"
    ↓
System collects cart items: {productId: qty}
    ↓
Calls: inventoryService.validateCartInventory(cartMap)
    ↓
For each item in cart:
    - Query Firestore for current stock
    - Compare: available >= requested
    - If any fail → return false
    ↓
If validation passes:
    - Create order
    - Deduct stock for each item
    - Clear cart
    - Show confirmation
    ↓
If validation fails:
    - Show error message
    - Don't create order
    - Keep cart intact
    - User can try again
```

### Database Schema
```
Firestore Collections:
├── products
│   ├── id: string
│   ├── name: string
│   ├── price: number
│   ├── stock: number ← INVENTORY FIELD
│   └── ...
│
└── shopping_carts
    ├── {userId}
    │   ├── items: array
    │   │   ├── id: string
    │   │   ├── productId: string
    │   │   ├── quantity: number
    │   │   └── ...
    │   ├── subtotal: number
    │   ├── lastUpdated: timestamp
    │   └── ...
```

## 📞 SUPPORT

If you encounter issues:

1. **Cart not loading:** Check `appInitializerProvider` is called in `main.dart`
2. **Stock showing wrong:** Verify Firestore `products.stock` field updated
3. **Button always disabled:** Check network connectivity and Firestore permissions
4. **Checkout still allowing out-of-stock:** Verify `validateCartInventory()` called in `_completeOrder()`

See detailed troubleshooting in: `CART_PERSISTENCE_AND_INVENTORY_WARNINGS_IMPLEMENTATION.md`
